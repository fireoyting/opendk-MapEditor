以下是对 DarkEden 客户端地图系统的核心技术梳理，专为计划开发地图编辑器、研究游戏底层架构的开发者整理：
一、坐标系统
客户端设计了三套各司其职的坐标体系，是地图编辑器开发的核心基础：
格子坐标系统（逻辑层）
地图最大尺寸为 256×256 个 sector（可理解为地图格子 / 地板单元）。每个 tile 贴图尺寸固定为 48×24 像素，贴图中心点为该 tile 的逻辑坐标原点，角色移动、寻路等服务端核心逻辑均基于此计算。
像素坐标系统（绘制层）
以像素为单位覆盖整个地图绘制区域，最大尺寸为 256×48 像素 × 256×24 像素，直接对应渲染时的实际位置，用于将 tile 贴图绘制到缓冲区。
视口坐标系统（显示层）
对应游戏窗口的显示区域（如 800×600、1024×720、1920×1080），设计上强制视口左上角与某一 tile 中心点对齐，避免地图平移 / 滚动时因半格偏移产生视觉抖动。
开发关键：需实现三套坐标的快速转换函数，例如：
鼠标点击屏幕坐标 → 视口坐标 → 像素坐标 → 格子坐标（用于选中 tile）
角色移动格子坐标 → 像素坐标（绘制 sprite）→ 视口坐标（调整镜头）
注：私服主流的大分辨率适配方案仅简单修改视口参数，未处理图像适配问题。

三、地图格式
1. 基础头信息
plaintext
版本地图头：地图版本号：地图名称：地图编码
tile 的 SPI 偏移值（4 字符）// 快速读表定位
objectimage SPI 偏移值（4 字符）// 快速读表定位
地图尺寸：最大 256×256 个格子（对应源码 sector 类）
2. Tile 数据结构
每个格子（tile）由 3 个字段按顺序拼接存储：
素材编号（2 字符）：标识图块素材，理论最大支持 65535 种；
通行标识（1 字符）：独立存储，便于寻路逻辑快速读取；
光照标识（1 字符）：预留字段，计划用于光照强度 / 动态光影（暂未启用）。
3. 物体与动态特效层
无数据压缩，每个对象记录tile列表：
plaintext
<物体和动态特效数组>
对象 遮挡格子数量 位置列表（索引: 坐标）
#路灯 35 0: (0,1), 1: (2,3), ..., 34: (25,88)
#树木 12 0: (10,20), 1: (12,22), ..., 11: (30,40)
#篝火动画 2 0: (5,5), 1: (15,15)
4. SPI 核心说明
SPI（Sprite Position Index，精灵图位置索引表）记录精灵图在打包文件中的起始偏移量：
tilespk 的 SPI：客户端读取地图时定位读取 SPK 文件；
objectimage 的 SPI：客户端读取地图时定位读取 ISPK 文件；
核心作用：基于早期 ODL 机制，通过 SPI 快速定位并按需读取图片，减少内存占用。
5. 开发避坑要点
开源 DK2 版本源码SPI默认仅使用 tile.spk 和 objectimage.ispk，编辑器开发中切勿随意引入 （客户端过图闪退多因 SPI 处理不当）；
动态元素通过编排动画包实现基础动画效果；
历史原因导致部分地图未严格分层，所有元素（互动 / 静态 / 动态）被塞入 TileLayer 或 ObjectLayer 并压缩，编辑器开发初期可仅按「是否为动态元素」做简单区分，优先保证呈现效果。
